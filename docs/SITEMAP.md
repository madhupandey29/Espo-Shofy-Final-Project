# Dynamic Sitemap Generation

This document explains the dynamic sitemap generation system implemented in your Next.js application.

## Overview

The sitemap is automatically generated and includes:
- ✅ All static pages (home, shop, blog, contact, etc.)
- ✅ Dynamic product pages from your API
- ✅ Blog post pages
- ✅ Category pages (if available)
- ✅ Proper SEO metadata (priority, change frequency, last modified)

## How It Works

### 1. Automatic Generation
The sitemap is automatically generated by Next.js when someone visits `/sitemap.xml`. It:
- Fetches all products from your API
- Includes all static routes
- Adds blog posts from your blog data
- Attempts to fetch categories if available

### 2. Caching & Performance
- **Revalidation**: 5 minutes (300 seconds)
- **API Caching**: Product and category data cached for 5 minutes
- **Error Handling**: Falls back to static pages if API fails

### 3. SEO Optimization
- **Priority Levels**:
  - Homepage: 1.0 (highest)
  - Shop pages: 0.9
  - Product pages: 0.8
  - Blog pages: 0.6-0.8
  - Other pages: 0.4-0.7
- **Change Frequency**: Optimized per page type
- **Last Modified**: Uses actual update dates when available

```env

NEXT_PUBLIC_SITE_URL=https://espo-shofy-final-project.vercel.app
NEXT_PUBLIC_API_BASE_URL=https://espobackend.vercel.app/api
```

### Customization

To modify the sitemap:

1. **Add new static routes**: Edit `SitemapManager.getStaticRoutes()`
2. **Change priorities**: Modify priority values in the manager
3. **Add new data sources**: Extend the manager class
4. **Modify caching**: Change `revalidate` value in `sitemap.js`

## API Integration

The sitemap automatically fetches:

### Products
- **Endpoint**: `/newproduct/view`
- **URL Pattern**: `/fabric/{slug}`
- **Fallback**: Continues without products if API fails

### Categories
- **Endpoint**: `/category/view`
- **URL Pattern**: `/shop-category?category={slug}`
- **Fallback**: Skips categories if not available

### Blogs
- **Source**: Local blog data file
- **URL Patterns**: 
  - `/blog-details/{id}`
  - `/blog-details-2/{id}`

## Troubleshooting

### Common Issues

1. **Empty Sitemap**
   - Check API connectivity
   - Verify environment variables
   - Check console logs for errors

2. **Missing Products**
   - Ensure products have `slug` field
   - Check API response format
   - Verify API endpoint accessibility

3. **Outdated Content**
   - Wait for revalidation (5 minutes)
   - Check `lastModified` dates
   - Verify API returns updated timestamps
